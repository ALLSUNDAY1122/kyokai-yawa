# Claude実装比較用プロンプト

あなたはフロントエンドとパーティゲーム演出に強いエンジニアです。
添付の `nomikai-arcade.html` は、スマホ1台で遊べる飲み会ゲーム集です。
既存の世界観、単一HTML構成、サウンド、振動、パーティクル、20本前後のゲームロジックは活かしつつ、さらに「実際の飲み会で迷わず使える完成品」にしてください。

## 実装してほしい改善

1. ホーム画面に「司会おすすめ」パネルを追加してください。
   - 現在のモードや直近に遊んだゲームを見て、次に遊ぶゲームを1つ提案する。
   - なぜそのゲームが向いているかを短い司会コメントで表示する。
   - 「この札で開演」「おまかせ再抽選」のような即開始導線を置く。

2. ゲーム一覧にモード切替を追加してください。
   - `全部`
   - `友人`
   - `恋寄り`
   - `即決`
   - モードに応じてゲーム札の表示を絞り込む。

3. 安全・境界確認モードを追加してください。
   - 恋愛質問、密着系、カメラ系、匿名投票系などの前に、全員がOKか確認する画面を挟む。
   - 「パス可能」「飲む量は自由」「撮影・カメラは保存しない」などのルールを明示する。
   - 一度確認したゲームは、そのページを開いている間は再確認しなくてよい。
   - ホームでON/OFFできるようにする。

4. 状態保存を追加してください。
   - プレイヤー人数
   - プレイヤー名
   - 今夜の戦績
   - サウンドON/OFF
   - モード
   - 境界確認ON/OFF
   - 直近に遊んだゲーム
   - `localStorage` を使い、失敗してもアプリが止まらないようにする。

5. 既存ゲームロジックは壊さないでください。
   - 個別ゲームのルールや演出は、必要な軽微修正以外は維持する。
   - とくにカメラ、モーションセンサー、長押し、投票、結果発表の動作を壊さない。

6. UI品質の基準
   - スマホ幅430px前後で最初の画面が気持ちよく見える。
   - ボタンは親指で押しやすく、テキストがはみ出さない。
   - 既存の「呑み処アーケード」世界観を崩さず、司会・札・提灯・判子の雰囲気に合う。
   - モード切替や安全設定は説明的すぎず、飲み会の司会道具として自然に見える。

## 成果物

`claude-nomikai-arcade-improved.html` という単一HTMLファイルを作成してください。
外部ライブラリは追加せず、現在と同じくHTML/CSS/JSを1ファイルにまとめてください。

## 比較時の評価ポイント

- 初見で「何から始めるか」が分かるか。
- 場が止まったときに司会パネルが助けになるか。
- 恋愛・密着・カメラ系で不快感を減らす配慮があるか。
- 元の派手さやワクワク感を落としていないか。
- スマホで片手操作しやすいか。
- 既存ゲームが壊れていないか。
